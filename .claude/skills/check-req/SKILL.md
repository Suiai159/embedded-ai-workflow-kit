# /check-req — 需求一致性检查

**防止代码与需求文档脱节。** 在编译前自动检查，发现不一致立即报警。

## 使用方式

```
/check-req                    # 检查当前项目
/check-req --strict           # 严格模式：任何不一致都报错
/check-req --fix              # 尝试自动修复（基于需求.md更新代码）
```

## 检查逻辑

### 1. 读取需求.md
提取关键参数：
- 时序参数（周期、频率、延时等）
- 硬件配置（引脚、电平、外设等）
- 阈值数值（温度、光照、距离等）
- 功能开关（使能/禁用、模式选择等）

### 2. 扫描源码
在以下位置查找实际使用的值：
- `#define` 宏定义
- `const` 常量
- 结构体初始化值
- 函数参数默认值

### 3. 比对规则

| 需求文档 | 代码实际值 | 状态 |
|----------|-----------|------|
| 呼吸周期 8s | `#define BREATHE_PERIOD_MS 8000` | ✅ 一致 |
| 呼吸周期 8s | `#define BREATHE_PERIOD_MS 10000` | ❌ 不一致 |
| PC13 低电平有效 | `GPIO_ResetBits(LED_PIN)` 亮灯 | ✅ 一致 |
| PC13 低电平有效 | `GPIO_SetBits(LED_PIN)` 亮灯 | ❌ 逻辑反了 |
| PWM频率 ≥100Hz | `#define PWM_FREQ 200` | ✅ 一致 |
| PWM频率 ≥100Hz | `#define PWM_FREQ 50` | ❌ 不满足要求 |

## 输出格式

```
## 需求一致性检查报告

### 检查概要
- 检查参数: 12 个
- 一致: 10 个 ✅
- 不一致: 2 个 ❌

### 详细结果

#### ✅ 一致参数
| 参数名 | 需求值 | 代码值 |
|--------|--------|--------|
| 呼吸周期 | 8000ms | BREATHE_PERIOD_MS = 8000 |
| PWM频率 | 200Hz | PWM_FREQ = 200 |

#### ❌ 不一致参数
| 参数名 | 需求值 | 代码值 | 位置 |
|--------|--------|--------|------|
| LED引脚 | PC13 | PA1 | led_service.h:15 |
| 呼吸曲线 | 正弦波 | 线性 | breathe_app.c:42 |

### 建议修复
1. 修改 `led_service.h:15`: `#define LED_PIN GPIO_PIN_13` → `GPIO_PIN_13`
2. 修改 `breathe_app.c:42`: 使用正弦表替代线性计算
```

## 触发时机

### 自动触发
- `/build` 前自动执行
- `/verify` 前自动执行
- `/flash` 前自动执行

### 手动触发
- 用户主动调用 `/check-req`

## 防呆机制

### 代码比需求新
如果代码修改时间 > 需求.md修改时间：
- 警告："代码比需求文档新，可能需求已过时"
- 建议："建议先运行 `/sync-req` 更新需求文档"

### 需求参数未找到
如果需求.md中的参数在代码中找不到：
- 警告："需求参数 'xxx' 在代码中未找到对应实现"
- 可能原因：
  1. 参数名映射关系丢失
  2. 该功能尚未实现
  3. 代码被误删

### 代码参数未找到
如果代码中使用的值在需求.md中找不到：
- 信息："代码使用参数 'yyy'，需求文档未定义"
- 建议："如需规范，请补充到需求.md"

## Entry Point

当调用 `/check-req`：

1. **读取需求**：解析需求.md提取所有可量化参数
2. **扫描代码**：遍历源码查找对应实现
3. **建立映射**：将需求参数与代码值关联
4. **执行比对**：逐条检查一致性
5. **生成报告**：输出检查结果
6. **决策**（如果是自动触发）：
   - 全部一致 → 继续执行原命令（build/verify/flash）
   - 存在不一致 → 报错停止，提示用户运行 `/sync-req` 修复

## 严格模式

使用 `--strict` 参数时：
- 任何不一致都视为错误
- 禁止继续编译/烧录
- 必须修复后才能继续

## 修复模式

使用 `--fix` 参数时：
- 基于需求.md的值自动修改代码
- 生成修改报告
- 用户确认后才写入
